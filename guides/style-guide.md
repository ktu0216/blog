# Saegida 블로그 스타일 가이드 (어투·문체)

> `sample/`의 기술 블로그 글에서 **어투·문체 패턴만** 추출했다(주제·내용은 무시).
> AI 작가 에이전트가 **이 파일 하나만 보고** 같은 톤으로 글을 쓸 수 있도록 정리.

## 0. 한 줄 요약
**독자에게 말 걸듯 질문을 던지고, "문제 → 왜 필요한가 → 정의 → 실제 사례(시행착오) → 단계별
정리 → 교훈" 순으로 친절하게 풀어주는, 정중하지만 친근한 기술 블로그 톤.**

## 1. 말투 / 어조 (레지스터)
한 글 안에서는 **반드시 하나로 통일**한다. 둘 중 택1:

- **기본(권장) — 정중한 `~습니다/합니다`체** + 군데군데 친근한 종결.
  - 예: "~할 수 있습니다.", "참고하실 수 있습니다.", "지금은 ~만 이해하시면 됩니다."
  - 친근함 가미: `~이죠.`, `~인데요.`, `~거든요.` 로 설명 끝 톤다운.
  - 독자 호명·기원: "~분이라면", "혹시 ~하시나요?", "~되었길 바랍니다."
  - 친근한 인사 도입 가능: "안녕하세요. ~팀 ~입니다!"
- **대안 — 평서 `~다/이다`체** (개인 회고·기술 르포·발표 톤).
  - 예: "데이터 분석은 선택이 아닌 필수다.", "결과물도 나쁘지 않았다."

> 과한 이모지·감탄은 자제(있어도 글당 1~2개). 가르치기보다 **함께 짚어보는 동행 톤**.

## 2. 문장 끝맺음
- 평서 `~습니다.`(또는 `~다.`)가 주력.
- **질문형으로 문단·소제목을 자주 연다**: "~할까요?", "~어떨까요?", "~해야 한다면?"
- 부드러운 단정: "~인 것이죠.", "~라는 뜻입니다.", "~면 됩니다."
- **짧은 문장으로 핵심을 못 박기**: "바로 이 문제를 해결하는 것이 RAG입니다."

## 3. 문장 호흡 / 길이
- 중간 길이 위주, **한 문단 2~4문장**.
- 긴 설명 문장 뒤엔 **짧은 정리 문장**으로 리듬을 준다.
- 개념 한 덩이 → 줄바꿈 → 예시 한 덩이. 빽빽한 장문 단락 금지.
- 나열은 **불릿/번호**로 끊어 가독성을 높인다.

## 4. 자주 쓰는 표현 / 어휘
- **전환·정리**: "즉,", "쉽게 얘기하면,", "정리하면,", "결국", "바로 이"
- **예시 도입**: "예를 들어", "~라고 해보죠.", "앞서 본 ~ 예시에서"
- **독자 끌기**: "한 번쯤 겪어봤을", "혹시 ~하시나요?", "~해보면 어떨까요?"
- **짚기·안내**: "먼저 ~부터 짚어보겠습니다.", "~를 살펴보겠습니다.", "지금은 ~만 이해하시면 됩니다."
- **강조 블록**: "유의사항", "핵심은", "핵심 설계 포인트는 다음과 같습니다."
- **용어 병기**: 한글(English) — "환각(Hallucination)", "청킹(Chunking)", "호출 제한(Rate Limit)"
- **인용부호 `'...'`** 로 개념·원칙을 살짝 강조 — "‘완벽함보다 빠른 검증’"

## 5. 도입부 패턴 (아래 중 1~2개 조합)
- **(a) 한 줄 정의/요약** + 이 글에서 다룰 것 안내 + **독자 타겟팅**:
  "처음 접하는 분이라면 전체 흐름을, 익숙한 분이라면 핵심 질문을 참고하실 수 있습니다."
- **(b) 공감 질문/상황으로 시작**: 독자가 겪는 불편·궁금증을 큰따옴표 질문으로 던진다.
  "“이번 주 미팅 일정은?”" / "“…매출 변화는 얼마인가요?”"
- **(c) 맥락·출처 밝히기**: 발표 기반·시리즈 N편 등. "~발표를 기반으로 재구성했습니다."
- **(d) 친근한 인사 + 가벼운 곁다리**(발음 이야기 등)로 진입 장벽을 낮춘다.

## 6. 마무리 패턴
- **교훈/원칙 한 줄로 압축**: "‘완벽함보다 빠른 검증’, 이 원칙이 핵심입니다."
- **행동/관점 제안 (독자 제안)**: 독자 유형별로 다음 행동·관점을 권한다.
  형식 → **"~분이라면 ~해보시길 권합니다."**
  - 예: "RAG 도입을 고민 중이라면, 먼저 프롬프트만으로 풀리는지부터 점검해보시길 권합니다."
  - 예: "비슷한 온콜 부담을 겪고 있다면, 가장 단순한 구조부터 시도해보시길 권합니다."
  - 변형: "~라면 ~부터 시작해보세요.", "~한 관점으로 바라보면 어떨까요?"
- **독자 기원**: "~되었길 바랍니다."
- **다음 글 예고/시리즈 연결**: "다음 콘텐츠에서는 ~를 다루겠습니다."
- 팀/저자 서명으로 닫기(가끔 구독·뉴스레터 유도).
- 마지막은 가볍고 긍정적으로.

## 7. 전개(구조) 패턴
- **문제 제기 → 왜 필요한가 → 정의 → 실제 사례(시행착오: 1차/2차/3차 시도) → 단계별 가이드 → 교훈.**
- **질문형 소제목**: "왜 ~가 필요한가?", "~란 무엇인가?", "~해야 한다면?"
- **하나의 비유/예시를 글 전체에서 반복 재사용**(일관된 멘탈모델 — 예: 캘린더 미팅 일정).
- 단계마다 **"유의사항"** 으로 실전 팁을 분리.
- **구체 수치**로 목표·성과를 표현: "5분 → 30초", "95% 절감", "90% 쿼리 3초 이내".

## 8. 그 외 눈에 띄는 특징
- **시행착오를 솔직하게** 드러낸다(1차 실패 → 2차 → 최종 해결).
- 기술 자랑보다 **실무 의사결정 기준** 제시("팀이 익숙한 환경이 생산성에 더 중요").
- 강조는 **굵게/인용부호**로, 과한 수식어는 자제.
- 영어 약어는 **반드시 풀어서 병기**하고 처음 한 번 설명.

## 9. DO / DON'T
| DO | DON'T |
|---|---|
| 질문으로 열기 · 일관된 예시/비유 | 한 글에 존댓말/반말 섞기 |
| 용어 한글(English) 병기 | 영어 약어를 설명 없이 남발 |
| 단계·번호로 구조화, 유의사항 분리 | 근거 없는 단정·과장 |
| 교훈 한 줄로 닫기 | 빽빽한 장문 단락 |
| 정중하되 친근하게 | 과한 이모지·감탄 |

## 10. AI 작가용 체크리스트
- [ ] 레지스터 통일(기본 `~습니다`체)
- [ ] 도입: 정의 또는 공감질문 + 독자 타겟팅 + (맥락/출처)
- [ ] 질문형 소제목 + 일관된 예시/비유
- [ ] 용어 한글(English) 병기, 인용부호로 핵심 강조
- [ ] "유의사항"·핵심 분리, 구체 수치 사용
- [ ] 마무리: 교훈 한 줄 + **행동/관점 제안("~분이라면 ~해보시길 권합니다")** + 독자 기원 + (다음 글 예고)
- [ ] 개인정보 없음 — `../CONTENT-RULES.md` 준수(공개 글은 더미값)
