# 블로그 SEO 가이드 — 네이버 · 구글 (2026년 6월 기준)

> 웹 리서치로 정리한 최신(2026-06) 블로그 SEO 규칙. AI 에이전트(writer/assembler)가
> 이 파일을 보고 검색 친화적으로 글·메타를 구성하도록.
> ⚠️ **주의**: 구글 항목은 공식 문서/평판 좋은 출처 기반(비교적 확실). **네이버 C-Rank·D.I.A.는
> 공식 비공개**라 업계 통설·관찰 기반(추정 포함). 수치는 가이드라인이지 절대값 아님.

## 핵심 멘탈모델 (2026)
- **구글**: *People-first + E-E-A-T(특히 Trust) + Information Gain*. 2026년 3월 코어 업데이트가
  **원본 경험·1차 데이터·직접 테스트·사례**에 가중. AI 스팸/검색엔진용 페이지는 강등.
- **네이버**: **C-Rank(블로거의 분야 전문성) + D.I.A.+(문서의 검색의도 부합·경험·체류)** 가
  검색 순위의 80%+를 좌우. 한 주제로 꾸준히, 직접 경험·사진이 담긴 글이 유리.
- 공통 결론: **키워드 채우기보다 "실제 경험이 담긴, 의도에 정확히 답하는, 한 주제 깊은 글".**

## 1. 제목 작성 규칙
**글자 수**
- **구글**: 50–60자 권장(잘림 방지). 구글은 글자수보다 **픽셀(~600px)** 로 자름. 한글은 폭이 넓어
  **30자 안팎**이면 안전.
- **네이버**: 검색결과·모바일 가독성 위해 **25~40자** 권장. 너무 길면 잘림.

**키워드 위치**
- **핵심 키워드를 제목 앞쪽(앞 1/3)** 에 배치. 제목은 D.I.A.가 검색의도 부합을 판단하는 1순위 요소.
- 핵심 키워드는 **제목에 1회**면 충분(중복·나열 금지).
- 검색 의도와 직결되는 구체 키워드 사용(예: "RAG 구현 6단계" > "RAG 이야기").

**피해야 할 단어/패턴**
- 과장·낚시: "충격", "역대급", "단돈", "완벽", "100% 보장", 과한 느낌표/이모지.
- 키워드 반복·나열("강남맛집 강남맛집 추천 강남맛집 베스트").
- 본문과 무관한 제목(클릭베이트) → 체류시간 하락 → 역효과.

## 2. 본문 구조
**길이**
- 블로그 글 **1,000~1,500단어(한글 ~1,500~2,500자) 이상** 권장. 단, **의도에 맞게** —
  "타이어 교체법"에 타이어 역사 3,000단어는 불필요. 깊되 군더더기 없이.
- 네이버는 **체류시간**(평균 2분 30초~3분+)을 중시 → 끝까지 읽히는 밀도가 핵심.

**도입–본론–결론 비율 (권장)**
- **도입 ~10–15%**: 공감 질문/문제 제기 + 이 글에서 얻을 것(독자 타겟팅). 핵심 키워드 자연 등장.
- **본론 ~70–80%**: 소제목으로 쪼갠 단계/주제. 예시·데이터·스크린샷.
- **결론 ~10–15%**: 요약 + 행동/관점 제안 + (다음 글).

**소제목 활용**
- **H2/H3로 구조화**(스캔 가능하게). 소제목에 **관련 키워드·변형**을 자연스럽게.
- 핵심 키워드를 **H2 최소 1개 이상**에 포함.
- 질문형 소제목("왜 ~가 필요한가?")이 검색의도·스니펫에 유리.
- 목록·비교표·체크리스트로 "정보 이득(Information Gain)" 신호.

## 3. 키워드 밀도와 배치
- **밀도는 매직 넘버 없음.** John Mueller: "키워드 밀도는 랭킹 시그널 아님." **1~2% 자연스러운 수준**이면
  충분하고, 그 이상 반복은 **스팸 필터 유발**.
- **밀도보다 배치가 중요.** 핵심 키워드를 다음 위치에 **1회씩**:
  - 제목 / 첫 문단(앞 100자 내) / H2 최소 1개 / (구글) 메타 디스크립션 / 이미지 alt·캡션 / URL 슬러그
- 본문엔 **동의어·연관어·변형**을 자연스럽게(토픽 커버리지). 한 키워드 도배 X.
- 한 글 = **하나의 핵심 키워드/의도**에 집중(주제 분산 금지).

## 4. 이미지
**개수**
- **구글**: 본문 길이에 맞게 의미 있는 이미지(스크린샷·도식). 장식용 남발 X.
- **네이버 D.I.A.+**: **직접 촬영/제작한 고화질 이미지 6~13장**이 유리(스톡 이미지보다 직접 자료).

**대표(썸네일) 이미지 규격**
- 네이버 게시글 썸네일: **1300×885** 또는 정사각 **720×720** 지원.
- 비율: PC **3:2**, 모바일 **1:1**, 외부 공유 **16:9** 가 자동 적용 →
  **16:9(1280×720)로 만들되 중요한 요소는 1:1(가운데) 안에** 들어오게 디자인.
- 구글/OG: 대표 이미지 **1200×630(1.91:1)**, `og:image` 지정.

**alt 텍스트 / 파일명**
- **구글**: `alt`에 **이미지 내용을 설명 + 키워드 자연 포함**(과도한 키워드 X). 파일명도 의미있게
  (`rag-pipeline.png` > `IMG_1234.png`).
- **네이버**: 에디터에서 **별도 alt 입력 기능은 없음.** 대신 **이미지 바로 위/아래 설명 문장**과
  파일명·문맥으로 이미지를 설명(접근성·문맥 신호).
- 용량 최적화(WebP/적정 압축)로 로딩 속도 확보(이탈 방지).

## 5. 해시태그 / 태그 활용
- **네이버**: 글당 **최대 30개**, 하지만 **관련성 높은 5~15개** 권장(무분별한 30개 채우기 역효과).
  - 핵심 키워드 + **연관·세부 키워드**(롱테일). 네이버 검색창 **자동완성·연관검색어** 활용.
  - 경쟁 큰 일반어보다 **구체적·저경쟁 키워드**가 노출에 유리.
  - 본문과 무관한 인기 태그(태그 어뷰징) 금지.
- **구글**: 해시태그는 랭킹 요소 아님. 대신 **카테고리·태그 분류 + 내부 링크(관련 글 연결)** 로
  토픽 클러스터를 만든다.

**✅ 글 끝에 연관 해시태그 줄 추가 (필수)**
- 모든 글은 **본문 맨 끝에 연관 해시태그 한 줄**을 붙인다(네이버 태그 + 가독성).
- **5~15개**, 본문 핵심 키워드 + 연관·세부(롱테일) 키워드 기반. `tags` 프론트매터와 일관되게.
- 형식: `#키워드1 #키워드2 #키워드3 ...` (공백 구분, 한 줄). 띄어쓰기 있는 말은 붙여서(`#기술블로그`).
- 본문에 안 나온 무관한 인기 태그 금지(태그 어뷰징). 경쟁 큰 일반어보다 **구체적 키워드** 우선.
- 예: `#Astro #GitHubPages #기술블로그 #정적사이트생성기 #블로그SEO #SSG`

## 6. 네이버 C-Rank · D.I.A.+ 가 좋아하는 패턴
**C-Rank (블로거 전문성)**
- **한 주제(카테고리)에 꾸준히** 발행 → 분야 전문성 축적(통상 **카테고리당 누적 50편+** 가 분기점).
- 주제 일관성, 활동 지속성(규칙적 발행), 사용자 반응(체류·공감·댓글의 *자연스러운* 누적).
- 잡다한 주제 분산 < 한 분야 깊이.

**D.I.A.+ (문서 품질·검색의도)**
- **검색 의도에 정확히 답하는** 구성(제목·도입에서 답을 약속하고 본문에서 이행).
- **1인칭 경험·직접 자료**(직접 찍은 사진 6~13장, 15초+ 동영상/모먼트), 구체적 수치·사례.
- **체류시간 2분 30초~3분+**(끝까지 읽히는 밀도·소제목·이미지 배치).
- 고유한 정보(다른 글에 없는 경험·디테일) = "정보 이득".

> 구글 대응(동일 철학): **E-E-A-T** — 저자 소개/이력, 원본 사진·영상, 1차 데이터·직접 테스트,
> 한 주제 종합 커버리지(토픽 권위). Trust(정확·정직·출처)가 가장 중요.

## 7. 어뷰징 / 저품질 판정 (피해야 할 것)
**콘텐츠**
- **키워드 과다 반복·도배**(제목/태그/본문), 키워드 스터핑.
- **유사 문서**: 타 블로그·뉴스와 내용/표현 거의 동일, 복붙, 살짝만 바꾼 글.
- **천편일률 템플릿**: 모든 글이 같은 소제목 수·본문 길이·마무리. "안녕하세요, 오늘은 ~에 대해
  알아보겠습니다"식 똑같은 도입/마무리 → 저품질 신호.
- **채우기(filler) 텍스트**, 알맹이 없는 장문, 검색엔진만 노린 글.
- **AI 그대로 양산** 자체가 문제는 아니나, **팩트체크·1인칭 경험·구조화·군더더기 제거**가 없으면 저품질.

**행위/기술**
- 매크로 공감·댓글·스크랩·조회수 조작, 불량/대량 IP, 품앗이.
- 대량 스팸 발행, 상업성 외부 링크 도배, 숨김 텍스트(글자색=배경색), 클로킹.
- 클릭베이트 제목(본문 불일치) → 체류 하락 → 역효과.

**결과**: 네이버 저품질(누락/순위 추락), 구글 Helpful Content·코어 업데이트 강등.

## ✅ 발행 전 SEO 체크리스트 (writer/assembler 용)
- [ ] 제목: 핵심 키워드 앞쪽 1회, 25~40자(한글), 과장어·반복 없음
- [ ] 도입 첫 문단에 핵심 키워드 자연 등장 + 글에서 얻을 것 명시
- [ ] H2/H3 소제목 구조화, 핵심 키워드 H2 1개+에 포함, 질문형 소제목 활용
- [ ] 키워드 밀도 1~2%(자연), 동의어/연관어로 토픽 커버
- [ ] 1인칭 경험·구체 수치·사례(정보 이득), 본문 1,500자+ 의도에 맞게
- [ ] 이미지: 직접자료 6장+(네이버), 대표 16:9(중요요소 1:1 내), alt/설명·파일명 의미있게, 용량 최적화
- [ ] 태그: 관련성 높은 5~15개(자동완성/연관어), 무관 태그 금지
- [ ] **글 끝에 연관 해시태그 한 줄** 추가(`#키워드 …`, tags와 일관)
- [ ] (구글) 메타 디스크립션 120~158자, 키워드 1회, URL 슬러그 키워드
- [ ] 유사문서·템플릿 천편일률·키워드 도배 아님, 조작 행위 없음
- [ ] **PII 없음** — `../CONTENT-RULES.md` 준수 / **톤** — `style-guide.md` 준수
