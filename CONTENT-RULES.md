# 블로그 콘텐츠 규칙 (Saegida)

작성: 2026-06-28 · 상태: **적용**

스택: **Astro + GitHub Pages** (정적·공개). **블로그는 공개 전용**이다.

---

## 1. 공개 전용 — 비공개는 블로그에 두지 않는다
- **블로그(`src/content/posts/`)에는 공개해도 되는 글만 작성한다.**
- **비공개·개인 내용은 블로그가 아니라 개인 문서 폴더(`code/ktu-docs`, Obsidian)에 저장한다.**
  → 비공개 글이 애초에 블로그 레포에 없으므로 레포가 공개여도 안전.
- `draft: true`는 "아직 발행 안 한 **작성 중** 글"용(프로덕션 빌드 제외). 발행 시 `draft: false`.

## 2. 개인정보(PII) — 공개 글에서 제외 또는 대체

**원칙: 공개 글은 처음부터 더미값으로 작성.** 실제값 넣고 나중에 지우는 방식 금지(이력·캐시에 남음).

### 제외/대체 대상
- 실명·가족·지인 이름 → 가명 또는 "사용자"
- 이메일/전화 → `user@example.com`, 더미 번호
- 집/회사 위치, 근무처 등 식별 정보
- 내부 IP·호스트명·포트·서버명 → `10.0.0.x`, `example-host`, 더미 포트
- 실제 내부 도메인·서비스 URL → `example.com`
- **비밀: API 키·토큰·비밀번호·시드·인증서 → 절대 게시 금지** (`<YOUR_API_KEY>` 플레이스홀더)
- 계정 ID·유저네임(노출 원치 않는 것)
- 미공개 사업/재무 수치(수익, 잔고, 거래 내역 등)
- 스크린샷·로그 속 위 정보 → 마스킹

### 대체 표기 표준
| 실제 | 공개 표기 |
|---|---|
| 이메일 | `user@example.com` |
| 도메인 | `example.com` |
| 사설 IP | `10.0.0.x` / `192.168.x.x` |
| 키/토큰 | `<YOUR_KEY>` / `***` |
| 경로 | `/home/<user>/...` |

## 3. 발행 전 체크리스트
- [ ] PII 스캔: 이메일/키/사설IP/실명/내부호스트 grep으로 점검
- [ ] 코드·설정 예시의 실제값 → 더미값으로 치환
- [ ] 스크린샷/로그 내 식별정보 마스킹
- [ ] 미공개 사실(재무·내부 구조) 제외 확인
- [ ] 외부 링크·인용 출처 정리
- [ ] (작성 중이면 `draft: true`, 발행 시 `draft: false`)

## 4. 작성 워크플로우
1. (작성 중) `draft: true` 로 초안 작성
2. PII 점검(위 체크리스트)
3. `draft: false` 로 전환 → 빌드/배포

> AI(어시스턴트)가 글을 쓰거나 변환할 때도 이 규칙을 따른다: **공개 글엔 실제 PII를 넣지 않고
> 더미값으로 작성.** 비공개·개인 내용은 블로그가 아니라 개인 문서 폴더(`code/ktu-docs`)에 둔다.
